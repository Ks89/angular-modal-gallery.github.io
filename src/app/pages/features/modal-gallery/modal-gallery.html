ModalGallery

The ModalGallery component is a collection of Images. It displays all (or some) images as thumbnails in a single row. Clicking on one of them, the image will be showed in full screen in a modal way.

In addition to this, ModalGallery provides navigation between modal images with both mouse and keyboard and it can enable/disable download features. To conclude, it is also possible to configure descriptions and to add a button to navigate to an external url.


Selector modal-gallery
Exported as modalGallery

Inputs

modalImages
Type: Array<Image> | Observable<Array<Image>>
An array of Image objects or an Observable or it that represent both the thumb gallery and the modal images. In fact, an Image is an object with both thumb and image source paths. For more info about Image, please check HERE.
imagePointer
Type: number

Optional parameter useful to open the modal gallery (passing a value >=0) showing the image with index = imagePointer.

I recommend to use this input, only if you want to build a custom gallery with "+n more" feature. Otherwise, you shouldn't pass this value (leave it undefined). For more information about that, I really encourage to see the official example to understand what I'm saying.
downloadable
Type: boolean

If true, you'll be able to download the current modal image always with keyboard shortcut (ctrl+s or cmd+s), but the download button will be displayed only if showDownloadButton will be true. Otherwise, all download features will be disabled. downloadable=false by default

Please be careful: on SmartPhones and Tablets you can't use keyboard shortcuts, so if you want to enable your users to download your images, please enable both downloadable and showDownloadButton to add the download button.
description
Type: Description interface
Interface to specify a custom description either a full string or an object with string properties.
  • full: it's the entire string. For instance, 'Description of Image 1/2 - My family - bla bla'
  • object used to compose the description, for instance in this way: `imageText2numberSeparator12beforeTextDescriptionbla bla`, where `2` is the index of the visible image, `12` the total number of images and `bla bla` the description passed thanks to the input Image object.
    • imageText (default is 'Image')
    • numberSeparator (default is '/')
    • beforeTextDescription: (default is ' - ')
showDownloadButton
Type: boolean

If both downloadable and showDownloadButton are true, the download button will be displayed in the modal view. Otherwise, not. downloadable=false by default
showExtUrlButton
Type: boolean

If true and the image url is defined, the external url button will be displayed in the modal view. Otherwise, not. downloadable=false by default

Image

The main object that represents an Image with both thumbnail, description and so on. To initialize ModalGallery you have to build an array/Observable of these objects.


img
Type: string

Mandatory source path of the modal image.
thumb
Type: stringoptional

Optional source path of the thumbnail image. If you don't provide this path, angular-modal-gallery will scale down the image with source path equals to img to display the thumbnail.
description
Type: stringoptional

Optional description text. Attention: this is not the full description, but only the last part, after image numbers. For instance: `Image 2/12 - description`
extUrl
Type: stringoptional

Optional external url path. You must use a valid url, because angular-modal-gallery will simply navigate to this url

ImageModalEvent

The payload of all events emitted by this library.


action
Type: Action

It's the enum Action
result
Type: number | boolean

The result/data of the emitted event. For example, it could be the index of the current image or a boolean to say if images are ready to display and so on...

Description

Interface to set a custom description.


imageText
Type: stringoptional

It's the beginning part of the description. By default its value is `Image`, but you can change the language specifying your value, for instance 'Immagine' (italian), 'Bild' (german), 'Imagen' (spanish) or also an empty string ''.

Example: `imageText 2/12 - bla bla`
numberSeparator
Type: stringoptional

It's the text between the current image number ad the total number of images.

Example: `Image 2numberSeparator12 - bla bla`
beforeTextDescription
Type: stringoptional

It's the text between the total number of images and the final description text

Example: `Image 2/12beforeTextDescriptionbla bla`
customFullDescription
Type: stringoptional

It's the full text description. If you set this value, all others will be ignored.

Obviously, if you want a dynamic description with the current image number and so on, you have to build it by yourself. Otherwise, you the other parameters of this object to customize the default description. have to build your description


Action

Enum that represents the source of an action. It can be one of these enum values:


  • NORMAL: the default value
  • CLICK: mouse click action
  • KEYBOARD: keyboard action (ctrl+s, cmd+s, esc, →, ← and so on)
  • SWIPE: swipe action
  • LOAD: data/images loaded by the library

Keyboard

Enum that represents keyboard's keycodes. It can be one of these enum values:


  • ESC: 27
  • LEFT_ARROW: 37
  • RIGHT_ARROW: 39
  • UP_ARROW: 38
  • DOWN_ARROW: 40
